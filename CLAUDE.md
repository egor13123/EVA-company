# EVA-company — рабочий контекст

## Главное: позиционирование сайта

**Сайт — это карточка компании, а не инструмент продаж.**

Цель сайта — представить компанию, её возможности и достижения. НЕ нужно:
- Агрессивных продающих формулировок («купите сейчас», «успейте до конца месяца»)
- Кнопок типа «Заказать», «Купить», «Корзина»
- Лидогенерационных трюков (поп-апы, таймеры, скидки)

Что нужно:
- **Кто мы** — представление компании, ценности, подход
- **Что умеем** — услуги, компетенции, специализация
- **Что уже сделали** — кейсы, цифры, портфолио
- **С кем работали** — клиенты, партнёры, опыт

Тональность: спокойная, уверенная, профессиональная. Сайт должен производить впечатление надёжного подрядчика, а не интернет-магазина.

## Дизайн-принципы

- **Чёрно-белый минимализм** — никаких ярких акцентов, цветных кнопок, градиентов
- **Типографика — главное:** засечный Cormorant Garamond для заголовков, Inter для основного текста
- **Анимации — плавные и спокойные** (~600–900 мс, ease cubic-bezier(0.22, 1, 0.36, 1))
- **Логотип** — ЕВА ТСК, в стиле классической рамки с ромбами по углам
- **Многостраничность** — 6 разделов: главная, о нас, наш опыт, услуги, сотрудничество, контакты

## Структура страниц

- `/` — главная (hero с фоновым фото, цифры, превью «о компании», превью услуг, CTA)
- `/about` — о нас (принципы, подход, цитата)
- `/experience` — наш опыт (200+ участий, 25+ исполненных контрактов)
- `/services` — услуги (главный слоган + сферы + процесс)
- `/partnership` — сотрудничество (партнёры + вакансии)
- `/contact` — контакты (юр.данные + форма «Заказать звонок»)

## Известные ограничения окружения

- Контейнер не имеет прямого доступа к `vercel.app` (host_not_allowed) — превью production-сайта проверяется только пользователем.
- Локальный git-прокси и GitHub MCP могут давать 403, если Claude GitHub App не установлен на аккаунте `egor13123`. Если push не идёт — попросить пользователя проверить установку приложения.
- Unsplash IDs нельзя предсказать «вслепую» — если нужно конкретное фото (например, Москвы), лучше попросить у пользователя ссылку или файл.
